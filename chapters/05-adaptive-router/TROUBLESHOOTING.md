# 配套代码 05：常见报错

## `/route` 总是选择默认模型

刚开始没有运行数据时，各模型分数接近，平局会优先使用 `DEFAULT_PROVIDER`。可通过真实调用、`/compare` 和 `/rate` 逐步产生证据。

## 只有一个模型可用

检查 `.env` 中两个 Key 是否都已替换示例文字。只有一个模型时，自动模式仍能工作，但没有模型间选择。

## `/rate` 提示没有等待评分的回答

只有普通对话成功后才会产生待评分项。`/route` 是本地预览，`/compare` 是临时比较，两者都不会成为 `/rate` 的对象。

## 评分后选择没有立刻变化

路由分数还包含可靠性、速度和探索项；一次评分不保证立刻翻转。输入 `/why` 查看全部分项，不要只看用户评分。

## `router_state.json` 损坏

程序会拒绝加载，不会静默覆盖。先执行 `/where` 或查看 `.env` 中的 `ROUTER_PATH`，把损坏文件改名备份，再重新启动。也可以在确认后使用 `/router-reset`。

## 路由状态会不会泄露问题正文

本章设计不保存完整提示词或回答，只保存任务分类、通用信号、成功/失败、响应时间、评分与分数。对话正文仍保存在 `conversation.json`，不要上传 `data/`。

## 自动模式选择了不符合直觉的模型

先输入 `/why`：

1. 查看任务分类是否符合预期；
2. 查看当前策略；
3. 查看各模型评分、可靠性、速度与探索分；
4. 必要时用 `/use` 临时手动指定；
5. 使用 `/rate` 留下反馈，或 `/router-reset` 清空实验数据。

## 模型请求失败

失败不会写入对话记忆，但会进入可靠性统计。检查 API Key、余额、网络、模型名称和服务端状态。一次临时故障不会让可靠性直接归零，因为路由器使用了平滑先验。

## 找不到 Python 包

确认已经激活本章虚拟环境并安装 `requirements.txt`。macOS / Linux 使用 `source .venv/bin/activate`；Windows 使用 `.\.venv\Scripts\Activate.ps1`。

## API Key 已经出现在截图或 GitHub

立即在对应平台撤销 Key 并重新创建。只删除截图或当前文件不够，因为 Git 历史和缓存可能仍保留旧内容。
