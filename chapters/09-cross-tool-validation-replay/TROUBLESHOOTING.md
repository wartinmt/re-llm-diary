# 配套代码 09：常见报错

## `validation_failed`

这不是 Python 异常，而是工作流拒绝宣布完成。输入演示中最常见的原因是索引记录的 SHA-256 与计划或磁盘文件不一致。先检查验证结果中的四个布尔项，不要直接把状态改成 complete。

## 两个工具都显示成功，为什么仍失败

工具成功只证明单步副作用发生了。文档和索引还必须满足跨工具约束：同一标题、同一路径、同一内容哈希。任何一项不一致，组合结果都不成立。

## 恢复后第一步为什么没有再次写文件

文档工具已经按原幂等键保存了持久回执。Runtime 重启后先查询工具侧回执，再重新验证文件内容；证据成立时跳过执行，只补做缺失的第二步。

## `同一幂等键已经绑定到不同回执`

调用者试图让同一业务动作代表两个不同效果。不要删除回执绕过检查。应确认这是同一个业务动作，还是确实需要一个新的幂等键。

## `已存在的执行计划不可变更`

同一个 `workflow_id` 已经有计划，但标题、路径、正文或步骤发生了变化。恢复必须使用原计划；真正的新版本应创建新的 workflow ID。

## `内容哈希不匹配` 或 `prev_hash 不匹配`

流水账可能被修改、删除或乱序。不要继续执行副作用。保留原文件，复制后分析；需要继续实验时使用新的数据目录。

## 回滚顺序为什么是索引在前、文件在后

正向依赖是“索引依赖文件”。若先删文件，索引会短暂指向不存在的目标。逆序补偿先撤掉引用，再移除被引用对象。

## 第二次回滚会不会再次删除

补偿动作也有稳定幂等键。重复调用会复用已有补偿回执，不产生第二次副作用。

## Windows 中文输出乱码

使用 Python 3.10 以上，并在 PowerShell 中执行：

```powershell
$env:PYTHONUTF8="1"
py main.py --self-test
```

仓库的 GitHub Actions 工作流已经固定设置 `PYTHONUTF8=1`。

## `python3` 或 `py` 找不到

macOS / Linux 先运行 `python3 --version`；Windows 运行 `py --version`。若均不存在，从 Python 官方渠道安装 Python 3.10 或更高版本。
