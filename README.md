# RE: 从零开始的大模型研究日记

> 一个普通开发者，从研究一个 DeepSeek 快捷指令开始，逐步走向开放式 AI Runtime。

这是《RE:从零开始的大模型研究日记》的官方配套工程。

这里保存每一阶段真实运行过的代码、实验记录和设计变化。每一章对应一篇文章、一个可运行示例，以及一个可以回到当时状态的版本标签。

## 从这里开始

### 01｜最小 DeepSeek 连续对话客户端

从一个 DeepSeek 快捷指令出发，通过 API 做出一个能够连续对话的终端客户端。

[查看第 01 章代码](chapters/01-deepseek-client)

### 02｜Token、上下文与调用成本

观察连续对话如何增长输入 Token，并拆分缓存命中、缓存未命中与输出成本。

[查看第 02 章代码](chapters/02-token-cost)

### 03｜跨启动的本地对话记忆

把只存在于程序内存中的消息保存为本地 JSON 文件，并在下一次启动时安全恢复。

[查看第 03 章代码](chapters/03-persistent-memory)

### 04｜接入第二个模型

在同一个持久对话客户端中接入 DeepSeek 与 GLM，手动切换，并使用相同上下文进行临时比较。

[查看第 04 章代码](chapters/04-second-model)

### 05｜第一次让程序自己选模型

用任务分类、成功率、响应时间和用户评分建立一个透明、可解释、会随运行记录更新的路由基线。

[查看第 05 章代码](chapters/05-adaptive-router)

### 06｜路由器开始算账，也开始怀疑答案

把真实 Token 价格、单轮预算和第二模型轻量验证纳入路由，分别记录主回答、验证与比较成本。

[查看第 06 章代码](chapters/06-cost-verification)

### 07｜程序崩了以后，AI 不该假装什么都没发生

用追加式任务流水账、事件序号与 SHA-256 哈希链保存一轮任务的真实阶段；已落盘答案只做本地恢复，远端结果未知时禁止自动重试。

[查看第 07 章代码](chapters/07-interrupt-recovery-replay)

### 08｜AI 说两遍没关系，工具做两遍就可能出事

用稳定幂等键、持久回执和动作流水账阻止副作用重复执行；无法查询结果时保存未知状态，补偿则作为新的可审计动作发生。

[查看第 08 章代码](chapters/08-tool-effects-idempotency)

### 09｜两个工具都成功了，结果还是错的

用不可变计划、现实状态后置验证、逆序补偿和证据驱动重放，让跨工具工作流不再把“每一步成功”误当成“整体正确”。

[查看第 09 章代码](chapters/09-cross-tool-validation-replay)

### 10｜我让 Runtime 接入一个陌生工具，它先拒绝了一次

用只读发现、manifest 与 AST 检查、隔离 probe / preview、显式准入和源码指纹，让新工具在执行前先建立可追溯的信任边界。

[查看第 10 章代码](chapters/10-plugin-discovery-admission)

## API Key 安全

不要将真实 API Key 写进源码或提交到 GitHub。

各章节会提供 `.env.example`。复制为 `.env` 后，再在本地填写自己的密钥。仓库根目录的 `.gitignore` 会统一忽略 `.env` 文件。

如果密钥曾经出现在公开仓库、截图、录屏或日志中，请立即撤销并重新创建。
